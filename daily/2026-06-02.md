## 二、Week 3 深度阅读：Safe Wallet 主方向

精读三个核心标准/SDK：

### ERC-4337 Account Abstraction
- 让账户变成可编程执行环境（不只是一个私钥）
- UserOperation → EntryPoint → Bundler → Paymaster 流程
- 对 SafePay 价值：session key、spending limit、policy 可放账户层
- **边界**：不直接解决 AI agent 安全，需要上层 guard/policy

### Safe Smart Account + Guards / Modules
- Safe 多签架构 + Module 系统（AllowanceModule、Zodiac 等）
- Guard 模式：交易执行前做安全检查
- Session Key 四约束全部有链上对应

### LI.FI Agent Integration
- Agent 如何拿到真实可执行的跨链交易请求
- x402 → LI.FI → Safe 的完整链路

→ 阅读摘要见 `references/week3-safe-wallet-reading-summary.md`
