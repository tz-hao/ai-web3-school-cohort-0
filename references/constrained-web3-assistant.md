# 受限 Web3 助手：DEX Swap 工作流

> AI × Web3 School · Agent Workflow 实践
> 核心命题：AI 可以做什么，绝对不能做什么，边界怎么划

---

## 场景

用户在 Uniswap 上用 100 USDC 换 ETH。AI 助手辅助完成，但不能接触私钥、不能绕过人工确认。

---

## 工作流全景

```
用户输入 → AI 准备 → 人工确认① → AI 检查链上 → 人工确认② → AI 生成交易 → 人工确认③ → 链上执行 → AI 验证
```

共 9 步，其中 **3 个人工确认点**，**4 个 AI 可自动完成的步骤**，**2 个链上操作**。

---

## 逐步拆解

### Step 1 · 用户输入（👤 人）

```
用户说："帮我在 Uniswap 上用 100 USDC 换 ETH，滑点 0.5%"
```

### Step 2 · 解析意图（🤖 AI 自动）

AI 从自然语言中提取结构化参数：

| 字段 | 值 | 来源 |
|------|-----|------|
| 协议 | Uniswap V2 | AI 推理 |
| 输入 Token | USDC | AI 解析 |
| 输出 Token | ETH | AI 解析 |
| 数量 | 100 | AI 解析 |
| 滑点 | 0.5% | AI 解析 |
| 网络 | **待确认** → 需要人工指定或从钱包读取 |
| 用户地址 | **待确认** → 需要钱包提供，AI 不能持有地址 |

**AI 做了什么**：自然语言 → 结构化参数。**不接触私钥、不发起交易。**

### Step 3 · 人工确认①（🛑 必须人）

AI 向用户展示解析结果，等待确认：

```
┌─────────────────────────────────────────┐
│ 🤖 AI 解析结果，请确认：                 │
│                                         │
│ 协议：Uniswap V2                         │
│ 交易：100 USDC → ETH                     │
│ 滑点：0.5%                               │
│ 网络：Ethereum Mainnet（需你确认）        │
│ 你的地址：（请从钱包粘贴，AI 不记录）      │
│                                         │
│ 确认无误？[确认] [修改]                    │
└─────────────────────────────────────────┘
```

**为什么必须人确认**：AI 可能解析错误（100 USDC vs 100 USDT），用户意图可能不完整（忘记说网络），这是第一道防线。

### Step 4 · 链上状态检查（🤖 AI 自动，只读）

用户确认后，AI 查询链上（只读 RPC / Contract Read）：

```python
# AI 调用的工具（只读，不需要签名）
1. 查 USDC balance（你的地址）
2. 查 USDC allowance（你的地址 → Uniswap Router）
3. 查 ETH 余额（你的地址）
4. 查 Uniswap 池子状态（USDC-ETH pair reserves）
5. 估算预期输出（基于 reserves + 滑点）
6. 查当前 gas price
```

**AI 做了什么**：只读查询，不写链。**不接触私钥。**

**查询结果示例**：
```
USDC 余额   : 500 USDC ✅（够用）
USDC Allowance : 0（需要先 approve）
预期收到   : ~0.045 ETH
Gas 估算   : ~$8.50（200,000 gas × 20 gwei）
池子流动性  : 充足 ✅
```

### Step 5 · 人工确认②（🛑 必须人）

AI 展示链上状态 + 风险分析，等待确认：

```
┌─────────────────────────────────────────┐
│ 🔍 链上状态检查结果：                    │
│                                         │
│ ✅ USDC 余额充足（500）                   │
│ ⚠️  需要先 Approve（allowance = 0）       │
│     → 这会额外消耗 gas                     │
│ ✅ 池子流动性充足                          │
│ ⚠️  预期收到 ~0.045 ETH（按当前价格）      │
│ 💰 预估总 Gas：~$15（approve + swap）      │
│                                         │
│ 📋 执行计划：                              │
│ 1. Approve Uniswap Router 使用 100 USDC   │
│ 2. Swap 100 USDC → ETH（滑点 0.5%）       │
│                                         │
│ 🛑 需要你确认后才继续                       │
│ [确认执行] [修改参数] [取消]                │
└─────────────────────────────────────────┘
```

**为什么必须人确认**：
- Allowance 是高风险操作——approve 无限额度可能被后续攻击掏空
- 价格可能在这个瞬间变动
- Gas 估算可能不准确
- 池子状态可能已变化（MEV / sandwich attack）

### Step 6 · 生成交易草稿（🤖 AI 自动，不签名）

用户确认后，AI 生成交易 calldata（不签名、不发送）：

```python
# AI 生成的交易草稿（JSON 格式，不含签名）
tx1_approve = {
    "chainId": 1,
    "to": "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48",  # USDC
    "data": "0x095ea7b3...",  # approve(Router, 100000000)
    "value": "0",
    "gasLimit": 50000,
    "gasPrice": "20000000000",
    "nonce": "待钱包填入",
    # ❌ 没有 signature 字段——AI 不能签名
}

tx2_swap = {
    "chainId": 1,
    "to": "0x7a250d5630B4cF539739dF2C5dAcb4c659F2488D",  # Uniswap V2 Router
    "data": "0x7ff36ab5...",  # swapExactTokensForETH(...)
    "value": "0",
    "gasLimit": 180000,
    "gasPrice": "20000000000",
    "nonce": "待钱包填入",
    # ❌ 没有 signature 字段——AI 不能签名
}
```

**AI 做了什么**：根据 ABI + 参数生成 calldata。**不持有私钥，不生成签名，不发送交易。**

### Step 7 · 人工确认③ + 签名（🛑 必须人 + 钱包）

用户在自己的钱包里逐笔确认：

```
┌─────────────────────────────────────────┐
│ 🔐 请在钱包中确认交易                       │
│                                         │
│ 第 1 笔：Approve USDC                     │
│ 合约：0xA0b8... (USDC)                    │
│ 方法：approve(spender, 100000000)          │
│ Spender：0x7a25... (Uniswap Router)       │
│ 金额：100 USDC                            │
│ Gas：~$5                                 │
│                                         │
│ [在 MetaMask 中确认] [拒绝]                │
└─────────────────────────────────────────┘

（第一笔确认后，等待链上确认...）

┌─────────────────────────────────────────┐
│ 🔐 请在钱包中确认交易                       │
│                                         │
│ 第 2 笔：Swap USDC → ETH                  │
│ 合约：0x7a25... (Uniswap Router)          │
│ 方法：swapExactTokensForETH                │
│ 输入：100 USDC                            │
│ 最少收到：~0.0448 ETH（含 0.5% 滑点）       │
│ Gas：~$10                                │
│                                         │
│ [在 MetaMask 中确认] [拒绝]                │
└─────────────────────────────────────────┘
```

**为什么必须人 + 钱包**：
- 签名发生在钱包本地，私钥从不离开钱包
- 用户必须在钱包弹窗中看清 `to`、`value`、`data`、`gas`
- AI 生成的 calldata 可能是恶意的（被篡改、幻觉、攻击）

### Step 8 · 链上执行（⛓️ 链）

交易发送到 mempool → 打包 → 确认。AI 监控状态。

### Step 9 · 验证结果（🤖 AI 自动，只读）

AI 查询交易回执，生成验证报告：

```
┌─────────────────────────────────────────┐
│ ✅ 交易完成                                │
│                                         │
│ Approve Tx：0xabc123...                   │
│ ✅ 成功 · Gas 用了 46,234                 │
│ 📎 etherscan.io/tx/0xabc123...            │
│                                         │
│ Swap Tx：0xdef456...                      │
│ ✅ 成功 · 收到 0.0453 ETH                  │
│ 📎 etherscan.io/tx/0xdef456...            │
│                                         │
│ 📊 实际效果：                               │
│ - 花费：100 USDC + ~$14.50 gas            │
│ - 收到：0.0453 ETH                        │
│ - 滑点：比预期多收 0.0003 ETH ✅           │
│                                         │
│ ⚠️  建议：revoke USDC allowance（安全习惯）  │
│ [帮我 Revoke] [稍后再说]                    │
└─────────────────────────────────────────┘
```

**AI 做了什么**：查询交易回执、对比预期和实际、生成报告、提醒安全操作。**只读。**

---

## 边界总结

| 步骤 | 谁做 | 权限级别 |
|------|------|----------|
| 解析用户意图 | 🤖 AI | 只读——自然语言处理 |
| 查询链上状态 | 🤖 AI | 只读——RPC / Contract Read |
| 生成 calldata | 🤖 AI | 只读——ABI 编码 |
| 展示风险和建议 | 🤖 AI | 只读——信息呈现 |
| 验证执行结果 | 🤖 AI | 只读——交易回执查询 |
| **确认解析结果** | 🛑 人 | 人工确认点 1 |
| **确认执行计划** | 🛑 人 | 人工确认点 2 |
| **钱包签名发送** | 🛑 人+钱包 | 人工确认点 3 + 钱包签名 |
| **持有私钥/助记词** | 🚫 AI 禁止 | 永远不 |
| **绕过钱包确认** | 🚫 AI 禁止 | 永远不 |
| **擅自发送交易** | 🚫 AI 禁止 | 永远不 |

---

## 风险和限制

| 风险 | 说明 | 缓解 |
|------|------|------|
| AI calldata 被篡改 | 恶意 prompt 注入可能让 AI 生成错误 calldata | 用户钱包弹窗是最后防线——必须人工核对 to/value/data |
| 链上状态过期 | AI 查询的价格在签名瞬间可能已变 | 交易中包含滑点保护，用户确认前刷新查询 |
| MEV / 三明治攻击 | 大额交易可能被抢跑 | AI 提醒用户使用 Flashbots 或私有 mempool |
| Approve 风险 | approve 无限额度后资产可被长期转走 | AI 建议 approve 精确额度，交易后提醒 revoke |
| AI 幻觉 | AI 可能编造不存在的合约地址 | 所有合约地址附 explorer 链接，用户交叉验证 |
| 用户盲签 | 用户习惯性点确认不看内容 | AI 在每次确认前用自然语言解释「你正在签什么」 |

---

## 这个设计的要点

不是「AI 能不能做某件事」，而是 **「AI 在什么条件下可以做，什么条件下绝对不能」**。

三个绝对不能跨越的红线：
1. 🚫 AI 不能接触私钥、助记词、API Secret
2. 🚫 AI 不能绕过钱包的确认弹窗
3. 🚫 AI 不能在没有人工确认的情况下发送交易

三个 AI 可以做的有价值的事：
1. ✅ 把自然语言变成结构化参数（节省手动填表时间）
2. ✅ 读取链上状态并生成人类可读的分析（比人手动查 Etherscan 快）
3. ✅ 生成 calldata + gas 估算 + 风险检查（减少手动计算错误）
